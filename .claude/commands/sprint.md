# /sprint — Start a new Prezva sprint

Run the following sequence to begin a sprint:

1. Run `git status` and report any uncommitted changes
2. Run `git pull` to get latest from main
3. Run `pnpm test` and report how many tests pass/fail
4. Read CLAUDE.md to refresh context
5. Report: current branch, last commit message, test status, any blockers
6. Ask: "Ready for task. What are we building?"
