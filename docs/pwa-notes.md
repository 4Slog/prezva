# PWA Service Worker Notes

## Why sw.js is committed to public/
next-pwa@5.6.0 is incompatible with Next.js 16 when using withPWA() wrapper.
Using withPWA() causes Vercel build failure:
  ENOENT: no such file or directory, lstat '/vercel/path0/.next/export-detail.json'

The service worker (public/sw.js) was generated by next-pwa during a local build
and is committed directly. Vercel deploys it as a static file.

## Cache strategies in sw.js
- / -> NetworkFirst (handles auth redirects)
- /agenda -> StaleWhileRevalidate, 1h TTL, max 50 entries
- /api/events/*/checkin -> NetworkFirst, 5s timeout (offline-first check-in)
- *.png|jpg|svg|ico -> CacheFirst, 1d TTL, max 100 entries

## How to update sw.js
Edit public/sw.js directly. Do NOT add withPWA() back to next.config.ts — breaks production.

## Future migration path
When @serwist/next has stable Next.js 16 support, migrate to that package.
